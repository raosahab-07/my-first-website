Case Study

Fintech Dashboard Redesign

A comprehensive UX overhaul for a Series B startup, streamlining complex data visualization to improve user adoption by 40%.

description Overview

The client, a fast-growing fintech startup, was struggling with a legacy dashboard that was difficult to navigate and slow to load. Users were overwhelmed by unstructured data, leading to high churn rates during the trial period.

My role was to lead the redesign from discovery to delivery, focusing on creating a modular design system and implementing a performant frontend architecture.

flag The Challenge

The primary challenge was dealing with technical debt while trying to innovate on the user experience. The existing codebase was a monolithic structure that made it hard to implement modern interactive charts.

  • close Slow initial page loads causing user frustration.
  • close Inconsistent UI components across different modules.
  • close Lack of mobile responsiveness for on-the-go traders.

Wireframes to High-Fidelity Mockups

check_circle The Solution

We decided to rebuild the frontend using Next.js to leverage server-side rendering for faster initial loads. I created a custom design system in Figma that mapped directly to Tailwind CSS utility classes, ensuring consistency between design and code.

For data visualization, we integrated Recharts with a custom wrapper to handle real-time websocket data, providing users with instant feedback on market changes.
